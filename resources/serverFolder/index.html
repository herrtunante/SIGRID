SIGRID — Systematic Iterative GRID

A global systematic grid of sampling points at 1×1 km spacing with nested sub-grids at wider spacings, built for Open Foris Collect Earth. Unlike degree-based grids, the distance between SIGRID plots stays very close to 1000 m at any latitude: plots are laid row by row (East to West, moving North to South), and the degrees that equal 1000 m are recomputed for the latitude of every row.

Download the grid by country

The grid is published pre-cut for each country, so you do not need to download tiles and clip them yourself. Every plot is tagged with the province (ADM1NM) and district (ADM2NM) it falls in, based on the UN 2023 administrative boundaries.

Browse the per-country ZIP files Country index for Google Earth (KMZ)

Each country ZIP contains:

  • <Country>_1x1km.csv — the full 1×1 km grid with the sub-grid columns (see data format) plus the province/district of every plot.
  • <Country>_2x2km.csv … <Country>_100x100km.csv — one CSV per sampling density. They are exact subsets of the 1×1 km file with consistent CE_IDs, so an assessment started at a coarse density can later be intensified with the finer files without re-sampling.
  • <Country>_plot_counts_by_district.csv — number of plots per province/district at each density.
  • A README describing how the files were generated.

For some countries the UN 2023 boundaries carry no district names; those are published as <Country>_grids_no_districts.zip and their plots have no province/district columns.

Collect Earth tip: Google Earth degrades with files above ~2,000 placemarks. For larger files use Tools → Utilities → Divide large CSV files in Collect Earth — aggregating by the ADM1NM or ADM2NM column works well.

Download the grid by 10×10 degree tile

The full grid is served as 10×10 degree tiles (the files listed on this page). The easiest way to locate a tile is through Google Earth: open the KMZ below, click the tile you need and use the download link in its popup. Tiles near the equator hold more than 1.2 million plots, so their ZIPs contain several CSV files.

Data format

Each tile CSV looks like this (sub-grid columns shortened for readability):

CE_IDyCoordinatexCoordinategrid_1grid_2grid_3…grid_50grid_100
2790_898659.994950429.9851232truetruefalse…falsefalse
2790_898759.994950429.9672064truefalsefalse…falsefalse
2790_898859.994950429.9492864truetruetrue…falsefalse
  • CE_ID is the unique, stable ID of the plot, composed of two numbers Y_X counted from the starting point of the grid generation at 85° North, 169° West. E.g. 2000_3300 means 2,000 km South and 3,300 km West of that starting point. The ID is identical across all densities and tiles.
  • yCoordinate / xCoordinate are the latitude / longitude of the plot centre (WGS84, EPSG:4326).
  • The grid_N columns tell which sub-grids the plot belongs to. Every plot belongs to sub-grid 1 (1×1 km); 1/4 of the plots belong to the 2×2 km sub-grid, 1/9 to the 3×3 km, 1/16 to the 4×4 km, and so on for spacings of 1, 2, 3, 4, 5, 6, 8, 9, 10, 12, 15, 16, 20, 25, 30, 50 and 100 km. Columns can be combined to derive other spacings — e.g. the 18×18 km sub-grid is the set of plots where both grid_9 and grid_6 are true.

Because the IDs are consistent across densities, a Collect Earth assessment can be intensified: start with the 8×8 km sub-grid and, as resources allow, continue with the remaining plots of the 4×4 km sub-grid, then 2×2 km, down to the full 1×1 km grid — without ever re-assessing a plot.
